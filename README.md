# PhotosToPanorama

A mobile-first web application that stitches multiple photos into a panorama image or a panorama pan video. Designed to run on Android via Capacitor, or in any modern browser.

---

## Introduction

Upload 2–8 overlapping photos taken left-to-right. The app sends them to a Python backend which:

- **To Pano** — stitches the photos into a single wide panorama JPEG using SIFT feature matching and homography chaining.
- **To Video** — pans across the stitched panorama and exports a browser-compatible H.264 MP4.

Both outputs can be downloaded directly from the browser.

| Layer | Technology | Purpose |

All libraries are **free and open-source**. No API keys required.

## How It Works

### Panorama stitching (`panorama.py`)

1. Decode each uploaded image and downscale to max 2000 px wide.
2. For every consecutive pair of original images, detect SIFT keypoints and match with FLANN + Lowe's ratio test.
3. Compute a homography H mapping `images[i+1]` → `images[i]` using RANSAC.
4. Chain the homographies: `H_to_0[i] = H[0] @ H[1] @ … @ H[i-1]` so every image is expressed in `images[0]` coordinate space.
5. Calculate the combined canvas bounding box, apply a translation offset, then warp all images onto the canvas in one pass.
6. Crop black borders left by perspective warping and encode as JPEG.

### Video generation (`video.py`)

1. Load the stitched panorama and scale to 720 px tall (1280 × 720 viewport).
2. Write frames using `imageio` + `libx264`: 2-second hold at the left edge, a smooth pan left → right at 8 px/frame, then a 2-second hold at the right edge.
3. The `-movflags +faststart` flag places the `moov` atom at the start of the MP4, enabling immediate playback and correct duration display in browsers.

### Frontend flow

1. User selects or drops 2–8 photos — thumbnails are shown with stable object URLs (memory-leak-free via `useEffect` cleanup).
2. On button click, photos are sent as `multipart/form-data` via Axios with `responseType: "blob"`.
3. The returned blob is displayed inline and offered as a download.
4. API errors (even from blob responses) are parsed by reading the blob as text and parsing the JSON `detail` field.

```bash
set PYTHONPATH=
python -m venv venv
venv\Scripts\pip install -r requirements.txt

venv/bin/pip install -r requirements.txt
```

```bash
cd frontend
npm install
```

## How to Run (Local Development)

### Step 1 — Start the backend

**Windows (PowerShell or CMD):**
```powershell
cd C:\Users\yqing\PhotosToPanorama
.\start-backend.bat
```

**Git Bash / Linux / macOS:**
```bash
bash start-backend.sh
```

The server starts at `http://0.0.0.0:8000`. Leave this terminal open.

Verify it is running:
```bash
curl http://localhost:8000/health
# {"status":"ok"}
```

### Step 2 — Start the frontend

Open a second terminal:
```bash
cd frontend
npm run dev
```

Open **http://localhost:5173** in your browser.

### Step 3 — Use the app

1. Tap or click the upload area and select **2–8 overlapping photos** taken left-to-right.
2. Click **To Pano** — wait a few seconds for stitching. The panorama appears below.
3. Click **To Video** — wait ~10–20 s. A 7–15 s pan video appears and can be played inline.
4. Click **Download** to save the result.

> **Tip:** Photos should overlap by ~30–40% and be uploaded in the same left-to-right order they were taken.

## Wrapping for Android (Capacitor)

Capacitor converts the React web app into a native Android APK.

### Prerequisites

- Android Studio installed with **Android SDK API 34+**
- **JDK 17** installed and `JAVA_HOME` set
- A USB-connected Android device (or AVD emulator) with developer mode on

### Build and sync

```bash
cd frontend

# 1. Set your backend URL (the phone must reach this address)
#    For LAN testing, use your PC's local IP (e.g. 192.168.1.100)
echo "VITE_API_BASE_URL=http://192.168.1.100:8000" > .env.production

# 2. Build the React app
npm run build

# 3. Sync the build into the Android project
npx cap sync android

# 4. Open Android Studio
npx cap open android
```

### In Android Studio

- **Run on device:** click the green **Run** button — installs and launches on connected device/emulator.
- **Build APK:** `Build > Build Bundle(s) / APK(s) > Build APK(s)` — output in `android/app/build/outputs/apk/debug/`.
- **Signed release APK:** `Build > Generate Signed Bundle / APK`.

### After each code change

```bash
npm run build && npx cap sync android
```

## Docker Deployment

Docker runs the full stack (backend + frontend) in two containers with a single command. No Python, Node, or venv setup needed on the host.

### Requirements

- Docker Desktop installed and running

### Start

```bash
cd C:\Users\yqing\PhotosToPanorama

docker-compose up --build
```

- Frontend available at **http://localhost**
- Backend API at **http://localhost:8000**
- nginx inside the frontend container proxies `/api/` → backend automatically.

### Stop

```bash
docker-compose down
```

### Run in background (detached)

```bash
docker-compose up --build -d

# View logs
docker-compose logs -f

# Stop
docker-compose down
```

### Cloud deployment (VPS / any server)

1. Copy the project to your server (git clone or scp).
2. Edit `frontend/.env.production` if you need to override the API URL, or leave empty to let nginx proxy it.
3. Run `docker-compose up --build -d`.
4. Point your domain to the server's IP. Configure SSL with Let's Encrypt (certbot + nginx).

### Rebuild after code changes

```bash
docker-compose up --build
```

Only the changed layer is rebuilt due to Docker layer caching.

## API Reference

| Method | Endpoint | Body | Returns |
|---|---|---|---|
| GET | `/health` | — | `{"status":"ok"}` |
| POST | `/api/panorama` | `multipart/form-data` files (2–8 images) | JPEG image |
| POST | `/api/video` | `multipart/form-data` files (2–8 images) | MP4 video |

Error responses return JSON `{"detail": "reason"}` with HTTP 400 or 422.
